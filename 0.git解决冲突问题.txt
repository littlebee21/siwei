0.git解决冲突问题


这时分两种情况。

1，你需要另一个分支的所有代码变动，那么就采用合并（git merge）。
    全部合并
2，你只需要部分代码变动（某几个提交），这时可以采用 Cherry pick。
    单独的合并



拉取开源新的版本：
    1，找到不一样的版本作为基准版本
        通过代码比较的方式找到
    2，合并开源代码的所有commit
        git reset 回退版本
        然后删除对应的git文件
        然后合并多个开源commit为1个commit

    3，将开源代码的所有commit
        git status会有很多的commit
        手动的将 所有的 commit形成一个patch



思考：
    按照工程化的代码递交，
    小的改动是对于参数的修改
    大的改动是对于组件和类的添加
    每个代码的改动针对的是功能的，添加，删除，修改
