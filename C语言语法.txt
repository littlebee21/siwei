        1,变量的定义：
            使用名字+符号 = 名字 =>真正的内容 //本质是一种语法的嵌套
            数组指针:   指针->数组
            指针数组:   数组->指针 
                功能+类型 = 翻译为指向数组的指针
                功能+类型 = 翻译为指向指针的数组
        2,变量初始化：
            指针操作：
                *在初始化过程中：类型标记
                
            
        3,变量的赋值：
            *在赋值过程中为：取值操作

    如何思考和识别C语言语法!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!
        0，找到对应的名称；
            每个名称只能指代一个数据结构
        1，识别名称指代的数据结构的，存储内容递归
            eg: int (*p)[N] //指向int[N] 的指针 
                int *p[N]   //指向int 类型的地址
                从右往左，确定定义的数据结构
        1.1 势必名称指代的数据结构指代的数据机构：
            也就是指针的概念，确认指针指向的数据类型：
            eg：函数指针：指向函数的指针
                数组指针：指向数组的指针
                结构体指针：指向结构体的指针

        2，对定义的数据内容进行运算
                随着运算的变化，实时的切换焦点到不同的内存空间
                p指针的运算，会受到指向类型的影响；
                p:
                    数组名字的计算：
                        int a[3][3]，
                            a[1][2] 想象a作为首地址，a+1/3的整个内存空间+2
        2.1 识别对应运算符会将对应的指针变量指向到哪个方向的内存空间
            eg： -> 会将结构体的指针 运算到  结构体的内容名称 ：    名称进行继续的计算
                []  会将数组的指针   运算到 数组的内容名称：        名称进行继续的计算
                eg:struct test test1{
                    char name[30];
                }*p;
                p->name       得到名字：    数组名字指代数组的首地址，运算是单个空间
                (p->name)[10] 得到内容：    得到11个内存空间内存储的值

        数组和指针的转化：
            int arr[][4][5] 不是三级指针 是 int (*arr)[4][5]
            int **arr[]   是三级指针

        指针类型翻译 struct * ： 指向struct类型的指针类型
        强转指针翻译（struct *）：强转为指向struct类型的指针类型
        指针类型涉及两块内存区域的操作，所以右侧指向的内存区域是形容词，左侧的p指针区域才是本体

        malloc的逻辑理解：
            分配一个内存空间
            然后将内存空间的的地址，转化成指向某种类型的地址类型，保存到一个地址值中
            ：表示操控的是两片区域的内存空间
