	[2] 计算机组成原理
		(1) 硬件
			1) 冯﹒诺依曼式体系结构 (哈佛结构)
				理论要点
				1、计算机硬件设备由存储器、运算器、控制器、输入设备和输出设备5部分组成。
				2、存储程序思想——指令和数据不加区别混合存储在同一个存储器中，数据和程序在内存中是没有区别的,它们都是内存中的数据
				3、提出计算机的数制: 二进制(0, 1)
									 最优的数制: e进制
									 三进制计算机(-1, 0, 1) 前苏联
				...
				冯·诺依曼又被称为“计算机之父”。
			2) 硬件组成
				a.输入设备：键盘、鼠标、麦克风、摄像头、扫描仪、游戏手柄 ...
				b.输出设备(IO设备): 显示器、投影仪、音响、打印机、串口 ...
				c.存储器(主存): 一类存储设备，用于存在存放了机器指令和操作的数据
				d.CPU 
					运算器 : ALU(算术逻辑单元) 执行算术运算(包括加减乘数等基本运算及其附加运算)和逻辑运算(包括移位、逻辑测试或两个值比较)
					控制器 : 从系统通电开始，直到系统断电,从PC指向的主存的地址处读取机器指令，分析指令，根据指令的指示，执行如下动作：
						取指 -> 译码 -> 执行 
					寄存器 : 是CPU内部用来存放数据的一些小型存储区域，用来暂时存放参与运算的数据和运算结果。速度快，数目少。
						特殊的寄存器叫PC（程序计数器）,其中有一个特殊的寄存器叫PC（程序计数器），在任何时刻PC都指向主存中的某条机器指令的地址。
				e. 总线：总线（Bus）是计算机各种功能部件之间传送信息的公共通信干线，它是由导线组成的传输线束。
					计算机的总线可以划分为:
						数据总线DB( Data Bus) ： 传输数据
						地址总线AB(Address Bus) : 数据地址
						控制总线CB(Control Bus) : 控制信号 (如读/信号, 片选信号，中断信号，复位信号等)
			3) 存储器分类
				a. 主存储器
					内存，是计算机的工作存储器。程序，待处理的数据和处理的结果都存储在内存中。
					内存读写速度比较快，但只能在加电时存储数据。
				b. 外存储器
					辅助存储器, 是用来长期保存数据的大容量存储器，目前多为磁存储器或光存储器。
					读写速度慢于内存，可以在不加电的情况下长期保存数据。
				c. 高速缓存 cache 
					CPU高速缓存（英语：CPU Cache，简称缓存）是用于减少处理器访问内存所需平均时间的部件。
					其容量远小于内存，但速度却可以接近处理器的频率。当处理器发出内存访问请求时，会先查看缓存内
					是否有请求数据。如果存在（命中），则不经访问内存直接返回该数据；如果不存在（失效），则要先
					把内存中的相应数据载入缓存，再将其返回处理器
					
			4) 计算机中的常用单位
				1 个元器件称为 1 比特（Bit） 或 1 位
				8 个元器件称为 1 字节（Byte）
				8×1024 个元器件就是 1024Byte，简写为 1KB；
				8×1024×1024 个元器件就是 1024KB，简写为 1MB；
				8×1024×1024×1024 个元器件就是 1024MB，简写为 1GB。
				8×1024×1024×1024×1024 个元器件就是 1024GB，简写为 1TB。
				8×1024×1024×1024×1024×1024 个元器件就是 1024TB，简写为 1PB。
				8×1024×1024×1024×1024×1024×1024 个元器件就是 1024PB，简写为 1EB。
				......
				PB 和 EB 这两个高级单位一般在大数据处理过程中才会用到。

		(3) Linux 系统简介 
			1) 起源与历史
				a. UNIX系统诞生：
					1969年，AT&T公司（美国电话电报公司）的贝尔实验室与MIT（麻生理工）合作开发了UNIX操作系统，
				旨在开发一个用于大型、多任务、多用户的操作系统。
				b. UNIX家族发展
					System V 系统五 : 
						有很多大公司在取得了Unix的授权之后，开发了自己的Unix产品，比如：
						国际商业机器股份有限公司的AIX、
						惠普公司的HP-UX					
					BSD (Berkley Software Distribution )： 加州大学伯克利分校伯克利软件实验中心
						
				c. Linux 横空出世  
					1991年，芬兰赫尔辛基大学的学生Linus Torvals为了能在家里的PC机上使用与学校一样的操作系统，
				开始编写了类UNIX。
					1991.8.25，Linus就在comp.os.minix新闻组中首次发布了一个Linux内核的公共版本 0.1版本 。
					1993年，大约有100余名程序员参与了Linux内核代码编写/修改工作，其中核心组由5人组成，
							此时Linux 0.99的代码大约有十万行，用户大约有10万左右。
					1994年3月，Linux1.0发布，代码量17万行
					
				d.  GNU
					GNU的英文全称是"GNU is Not Unix"（递归缩写，程序员的幽默），又称革奴计划
				是由Richard Stallman在1983年9月27日公开发起的。它的目标是在UNIX基础之上创建一套
				完全自由的操作系统。
					1990年，GNU计划已经开发出的软件。 
					1991年，林纳斯·托瓦兹编写出了与UNIX兼容的Linux操作系统内核并在GPL条款下发布。
					1992年，Linux内核与其他GNU软件结合，完全自由的操作系统正式诞生。 该操作系统往往被称为“GNU/Linux”或简称Linux。
				e. GPL 	
					GPL（GNU General Public License）是GNU通用公共授权（许可证）。	
					所有GNU的软件都遵循 GPL 条款。
					GPL力图保障你分享和修改某程序全部版本的权利——确保自由软件对其用户来说是自由的。

		(4) Linux体系结构	
			1) 内核 Kernel 
				a. 内核被称作为操作系统核心:
					Linux系统内核大概可分为：内存管理部分、进程管理部分、设备驱动管理部分、文件系统部分、网络管理部分;
				b. Linux内核版本
					第一种：由三个数字组成：r.x.y
						r：目前发布的内核主版本
						x：偶数表示稳定版，奇数表示测试版
						y：错误补丁修补次数
							主版本号在内核结构有较大变化时才会变更。
					第二种：major.minor.patch-build.desc
						1.major：主版本号
						2.minor：次版本号，新增功能时会发生变化；偶数表示稳定版，奇数表示测试版
						3.patch：对次版本的修订次数
						4.build：表示编译（微调）次数，每次编译可能对少量程序做优化修改但一般没有大的功能性变化
						5.desc：描述当前版本的信息。例如一些内核版本中有SMP，表示多对称处理器（Symmetric MultiProcessing）。
						查询当前Linux版本信息

				uname -a ： 查看内核版本
					
			2) Shell
				a. 历史  
						在早期计算机没有图形化界面的年代，如果我们需要与操作系统进行交互，是无法使用鼠标进行操作的，
					而是在终端的界面内输入相关的指令让计算机完成相应的操作。
						但是，使用鼠标进行系统操作的效率是十分低下的。而且在某些特殊情况下（例如图形化界面全面崩溃），
					使用指令似乎是唯一一种与操作系统打交道的方式。因此，Linux系统和其他的一些系统仍然保留用户通过指令
					与操作系统交互的方式。
						微软的DOS系统的指令依然可以在windows使用
						运行 cmd 或者 powershell
							定时关机 : shutdown -s -t time(单位为秒)
							取消关机 : shutdown -a
							滑动关机 ：slidetoshutdown
											
				b. 功能  
					Shell，中文翻译为“壳”（相对于kernel“核”），为了保护内核，隔离用户与内核，我们引入了Shell。
					①　Shell是命令行解释器。
					②　将用户命令翻译为System理解的指令，实现用户与系统的交互 (内核的守护天使、负责完成用户与内核之间的交互)。
					③　当需要重复执行若干命令，可以将这些命令集合起来，加入一定的控制语句，编辑成为Shell脚本文件(批处理文件)，交给Shell批量执行。 
				
				c . Shell的执行过程
					Step1:  User 输入文本命令
					Step2:  Shell将命令或按键转换(翻译)成内核理解的指令
					Step3： System做出相应，控制相关硬件。
					Step4： Shell将执行结果返回给User
					
				d. shell 的种类 
					1.Bourne Shell（简称sh）：
							由AT&T贝尔实验室的Bourne开发，是Unix系统上的第一个Shell程序，一经问世就成为了工业标准。
						现在几乎所有的Unix/Linux系统都支持它。不过sh的功能较薄弱，且没有历史记录等重要功能，
						因此目前大多数操作系统将其作为应急Shell使用。
					2.C Shell（简称csh）：
						由加利福尼亚大学伯克利分校研发，最初的研发目的是为了改进sh的缺点。由于其Shell脚本的风格接近
						C语言因此最初受到广大C语言用户喜爱。
					3.Korn Shell（简称ksh）：
						由David Korn开发，最初是为了解决sh的用户交互问题，使用ksh需要许可证，因此应用并不广泛。
					4.Bourne Again Shell（简称bash）：
						由AT&T贝尔实验室的Bourne开发，sh的增强版。随着bash的不断完善，它已经成为了最流行的shell。
					5.Debian Almquist Shell（简称dash）：
						比bash小，只需占用较小的磁盘空间，而且需要的库更少。不过相对于bash功能也更少。一般可以用于嵌入式领域。
					......
						查看当前系统可使用的shell：指令 cat /etc/shells   
						查看自己的终端的Shell类型：指令 echo $SHELL

			2) 命令提示符  
				Shell命令提示符标识了命令行的开始。用户在提示符后面输入一条命令并按【Enter】键。
			命令提示符格式：
				fs@ubuntu:~$
				|	|	  |
				|	|	当前目录路径
				| 主机名
			用户名
				$	表示当前用户为普通用户
				#	表示当前用户为超级用户(root 系统最高权限)
				
				root@ubuntu:/home/fs#
				
			3) shell 命令格式
				命令格式三要素： 命令名称、选项、参数。令名称是必须的，而可选项和参数是可以省略的。
				
				$Command  [-Options]  Argument1  Argument2  ... 
				指令		选项	 	参数1		参数2
				说明:
					Command：Shell命令或程序，区分大小写。日期的指令时date
					Options： 命令选项，用于得到命令更多特殊功能或改变命令执行动作的类型，由 ”-”引导，可以同时带多个选项。
				
				示例: 
					fs@ubuntu:~$ date		022年 01月 11日 星期二 16:48:18 CST
					fs@ubuntu:~$ clear		清屏
					fs@ubuntu:~$ ls			显示当前目录下所有文件
					fs@ubuntu:~$ ls -l		显示当前目录下所有文件及属性
					fs@ubuntu:~$ ls -a		显示当前目录下所有文件包括隐藏文件(Linux 下以"."开头的文件都是隐藏文件)
					fs@ubuntu:~$ ls -l -a	包含多个可选项
					fs@ubuntu:~$ ls -la		包含多个可选项
					
					shutdown [OPTION]...  TIME [MESSAGE]
					可选项: 
						-r 	重启
						-h  关机
						-c 	取消关机或重启 不需要指定TIME,可以添加 MESSAGE
					参数: 
						TIME 			时间(分钟)
						MESSAGE(可选) 	一条消息(消息会被广播给所有用户)
						
					fs@ubuntu:~$ sudo shutdown -h 20 "Bye bye"
					fs@ubuntu:~$ sudo shutdown -c "I will come back"
					fs@ubuntu:~$ sudo shutdown -r 10
					fs@ubuntu:~$ sudo shutdown -h/-r now		立刻关机
