前端项目框架对比


全局变量或者长周期
    1，路由 router    ：找到路由然后将内容分为多个页面划归模块
    2，入口 index
    3，模块：类
        模块通信： 类的通信
        模块调用： 依赖注入
    4，配置文件
    5，请求工具


组织方式：
 主模块  
    从模块 
        组件
            html
            css
            js
                类
                    函数

输入和输出方式
    界面输出
    resful输出
    通过调试工具可以进行查看


每个功能都对应有对应的页面
每个功能有自己的模块
通过模块可以定位到对应的三大模块



测试方法：
 

前端项目的意义：项目计划书
    1，不断的增加功能，不一定好看但是一定要多
    2，不断的添加项目的功能



框架的结构：
    1，统一的main函数流程
        初始化
        事件监听机制
    2，统一的对象管理，vue的实例构造，java的依赖容器
    3，维护的全局对象功能，
        解决竟态问题
        解决存储问题
        解决互斥和竞争问题


对象的构造方式，先创建对象，然后通过函数注入的方式对对象的功能进行扩充
可以看作式 js 对html文件的反向控制读写，
继承的实现，
    儿子吃掉爸爸的实现
    儿子挂在爸爸下面的实现
