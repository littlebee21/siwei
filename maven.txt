    协议
        约定文件结构
            /src/main/java
                源码目录
            /src/main/resources
                项目所需资源
            /src/test/java
                测试代码
            /src/test/resources
                测试所需资源
            /target
                打包输出目录
            /target/classes
                编译输出目录
            /target/test-classes
                测试编译输出目录
        3套生命周期
            clean：清理项目
                pre-clean
                clean
                    清理上一次构建生成的文件
                post-clean
            default：构建项目
                validate
                    验证工程是否正确，所需的信息是否完整
                compile
                    编译项目源码
                test-compile
                    编译项目的测试代码
                test
                    使用单元测试框架运行测试
                package
                    打包
                install
                    将包安装到本地仓库
                deploy
                    将包复制到远程仓库
            site：建立项目站点
                pre-site生成项目站带你之前需要完成的
                site 生成项目站点文档
                post-site 生成后需要完成的任务也
                site-deploy 项目站带你发布到服务器
    maven原理
        如何解决版本冲突
            1.先定义者优先原则
            2.路径近者优先原则
                直接引入
            排除依赖
                1.通过标签排除
                2.锁定版本
                    第一步：在dependencyManagement标签中锁定依赖版本
                        该标签只负责锁定版本并不会引jar包
                    第二步：在dependencies标签声明需要导入的jar包，无需再次声明版本
        仓库
            本地仓库（本机）
                构建maven项目时，首先会从本地仓库查找资源，找不到的话再从远程仓库下载
            远程仓库
                私服（局域网）私有仓库
                    优点
                        节省自己的外网带宽
                        加速maven项目构建
                        部署自己内部的第三方构件
                        提高稳定性，增强控制
                        降低中央仓库的负荷
                    Nexus仓库类型
                        1.hosted：宿主仓库
                            releases：发布版稳定的
                            Snapshot:快照，测试版未发行
                        2.proxy:代理仓库
                            central:中央仓库
                        3.group:仓库组
                        4.Virtual虚拟
                        这两个版本是最常用的
                中央仓库（外网）
                中国地区的镜像
